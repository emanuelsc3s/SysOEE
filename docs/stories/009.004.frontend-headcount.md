# Story 009.004: Frontend — Configuração de Headcount por Linha

## Status
Draft

## Story

**Como** Gestor PCP,
**Eu quero** definir para cada linha de produção a lotação, o cargo e a quantidade desejada de operadores,
**Para que** o sistema compare automaticamente o headcount planejado com os colaboradores ativos e evidencie déficits de pessoal.

## Acceptance Criteria

1. Seção "Headcount" acessível dentro da tela de edição do plano (`/pmp/:pmp_id`) como aba ou seção separada
2. Lista de headcounts configurados por linha: Linha | Lotação | Cargo | Desejado | Atual | Gap | Ações
3. Botão "Adicionar Headcount" → formulário com:
   - Select "Linha" (`tblinhaproducao`)
   - Select "Lotação" (valores distintos de `tbfuncionario.lotacao` onde `ativo = true`) — busca por texto
   - Select "Cargo" (filtrado pela lotação selecionada, valores distintos de `tbfuncionario.cargo`) — busca por texto
   - Input "Quantidade Desejada" (inteiro positivo)
4. Ao selecionar Lotação + Cargo, exibir imediatamente o `headcount_atual` calculado de `tbfuncionario`
5. Campo "Atual" exibido em tempo real sem necessidade de salvar
6. Coluna "Gap" exibida com sinalização visual:
   - Gap ≥ 0 (desejado ≤ atual): verde ✓
   - Gap < 0 (déficit): vermelho com número negativo (ex: `-2`)
7. Permitir múltiplos registros por linha (ex: 2 Operadores de Envase + 1 Técnico de Qualidade na mesma linha)
8. Cada registro pode ser editado e removido independentemente
9. Remoção com confirmação antes de executar

### Busca de headcount_atual em tempo real
```typescript
// Ao selecionar lotacao_id e cargo_id
const buscarHeadcountAtual = async (lotacao_id: number, cargo_id: number) => {
  const { count } = await supabase
    .from('tbfuncionario')
    .select('*', { count: 'exact', head: true })
    .eq('lotacao_id', lotacao_id)
    .eq('cargo_id', cargo_id)
    .eq('ativo', true)
    .is('dt_rescisao', null);

  return count ?? 0;
};
```
